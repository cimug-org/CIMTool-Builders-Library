# jpa-rdfs.xsl

This builder currently ships with CIMTool.

## Builder Description

The **[jpa-rdfs.xsl](jpa-rdfs.xsl)** builder generates a [Java](https://www.java.com/en/download/help/whatis_java.html) source file containing RDFS-schema-aware entity classes annotated for [Jakarta Persistence 3.1](https://jakarta.ee/specifications/persistence/3.1/) (Jakarta EE 10). Unlike the companion **[jpa.xsl](../jpa/builder.md)** builder which targets JPA 2.2 and uses the legacy `javax.persistence.*` namespace, this builder uses the updated `jakarta.persistence.*` namespace introduced in Jakarta EE 9 and is compatible with Hibernate 6.x and other Jakarta EE 10-compliant persistence providers, as well as Jakarta EE application servers such as WildFly, Payara, and WebLogic.

A further key distinction from **jpa.xsl** is the RDFS-schema-aware primary key strategy. For all profile classes in the `IdentifiedObject` hierarchy, `mRID` is used as the natural `@Id` — a `VARCHAR(100)` string key that mirrors the primary key generated by the companion **[sql-rdfs-ansi92.xsl](../sql-rdfs-ansi92/builder.md)** builder. For non-`IdentifiedObject` classes (such as `CurveData`, `RegularTimePoint`, and `TapChangerTablePoint`), a surrogate UUID `id` field is emitted as the `@Id` together with a heuristic `@UniqueConstraint` on the `@Table` annotation spanning all required columns, ensuring structural alignment with the SQL DDL schema produced by the companion builder.

The builder supports a comprehensive set of modern Java type mappings: `LocalDate`, `LocalTime`, `LocalDateTime`, and `OffsetDateTime` for date and time CIM datatypes, and `Double` for all numeric CIM types to preserve measurement value precision across the full CIM range.

This builder is designed to be used in conjunction with the companion **[sql-rdfs-ansi92.xsl](../sql-rdfs-ansi92/builder.md)** and **[csharp-ef-rdfs.xsl](../csharp-ef-rdfs/builder.md)** builders. Popular JPA implementations compatible with the generated entities include [Hibernate](https://hibernate.org/) and [EclipseLink](https://www.eclipse.org/eclipselink/).

## Builder NTE Configuration

Given this builder ships with CIMTool the below table highlights the existing NTE (Name/Type/Extension) settings for the builder.  We strongly recommend that you do not import customized versions of this shipped builder over the original in your local CIMTool installation. Doing so will force regeneration of any existing builder-generated artifacts for projects in your workspace that have this builder enabled.

The recommended best practice is to rename the builder file  (from ```jpa-rdfs.xsl``` to ```custom-jpa-rdfs.xsl``` for example) before importing. On import this will be interpreted as a new builder and you will be allowed to enter values in the Type and Extension fields.

>*NOTE: </br>CIMTool requires that file extensions be unique and will prevent you from entering an extension already assigned to a builder. This is because an artifact's name is derived by concatenating the base name of the CIMTool ```.owl``` profile with the file extension assigned to the builder. Therefore, a unique file extension must be assigned to each builder when imported. The file extension for a builder can be modified later from within the "Maintain XSLT Transform Builders" screen.*

A blank Includes File Name row indicates that no XSLT include file is required for this builder.

| Field | Value |
|---|---|
| Builder XSLT Name: | `jpa-rdfs.xsl` |
| Includes File Name: |  |
| Transform Builder Type: | `TEXT` |
| File Extension of Generated Files: | `jpa-rdfs.java` |
